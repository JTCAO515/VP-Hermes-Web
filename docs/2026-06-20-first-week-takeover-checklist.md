# VisePanda 接手首周清单

## 1. 文档目的

这份文档不是长期 roadmap，而是帮助新接手开发者在第一周建立正确的理解顺序，避免一上来就做高风险动作。

## 2. 接手前准备

- 先读 `HANDOFF.md`
- 再读 `docs/2026-06-20-engineering-handoff-notes.md`
- 再看 `README.md`
- 再看 `CHANGELOG.md`
- 确认当前部署仍是 `Vercel`

## 3. 第 1 天

- 本地跑起项目
- 跑 Python 与 Node 测试
- 只阅读主链路文件，不急着改

## 4. 第 2-3 天

- 对照线上站点走核心用户路径
- 记录文档与现实不一致的地方
- 区分主链路与兼容层

## 5. 第 4-5 天

- 只做小范围修复
- 先补测试，再动代码
- 优先用户可感知问题

## 6. 这一周不要做什么

- 不要先重构 `web/app.js`
- 不要先改部署
- 不要先删除 `static/*` 历史兼容层
- 不要未经验证修改 auth/chat 核心链路
